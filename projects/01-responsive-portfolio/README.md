# Responsive Portfolio Page

Build a personal portfolio page using only HTML and CSS (no JavaScript). This project focuses on semantic markup, modern CSS layout techniques, responsive design, and accessibility best practices.

## Features to Implement

- **Responsive navigation** — a top nav bar that collapses or adapts gracefully on smaller screens
- **Hero section** — a prominent banner area with a headline, tagline, and call-to-action link
- **Project cards grid** — a grid of cards showcasing portfolio projects, each with an image, title, and short description
- **About section** — a brief bio/introduction section with text and an optional profile image
- **Contact form (styling only)** — a styled contact form with fields for name, email, and message (no backend required)
- **Footer** — a site footer with social links and copyright information

## Technical Requirements

- Use only HTML and CSS (no JavaScript)
- Use semantic HTML elements (`<header>`, `<nav>`, `<main>`, `<section>`, `<article>`, `<footer>`)
- Implement a mobile-first responsive layout using CSS Grid and/or Flexbox
- Use CSS custom properties (variables) for colors, fonts, and spacing to enable easy theming
- Add smooth CSS transitions or animations (e.g., hover effects on cards, fade-in on hero)
- Ensure the page passes basic accessibility checks (proper heading hierarchy, alt text, focus styles, sufficient contrast)
- The page should look good on mobile (320px), tablet (768px), and desktop (1024px+) viewports

## Stretch Goals

- **CSS-only dark mode toggle** — implement a dark/light theme switcher using only CSS (e.g., with a hidden checkbox and `:checked` selector)
- **Sass refactor** — refactor the CSS into Sass with partials, variables, mixins, and nesting
