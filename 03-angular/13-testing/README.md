# Testing

## Learning Objectives
- Write unit tests with Jasmine using describe, it, expect, and beforeEach
- Configure TestBed for Angular-specific testing
- Test components with both shallow and deep rendering approaches
- Test services with and without dependencies
- Mock dependencies using spies, stubs, and jasmine.createSpyObj
- Handle asynchronous tests with fakeAsync, tick, and waitForAsync
- Apply integration testing patterns
- Set up end-to-end tests with Cypress or Playwright

## Key Concepts
Testing ensures that Angular applications work correctly and remain stable as they evolve. Unit tests verify individual components and services in isolation using Jasmine as the test framework and TestBed as the Angular-specific testing utility. Shallow tests render a component without its children, while deep tests include the full component tree. Mocking replaces real dependencies with controlled substitutes, and Angular provides utilities for testing asynchronous code that would otherwise be difficult to test deterministically. End-to-end tests validate complete user workflows through the running application.
