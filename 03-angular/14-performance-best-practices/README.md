- Apply pure pipes and OnPush change detection for performance gains
- Implement virtual scrolling with Angular CDK
- Set up server-side rendering with Angular Universal/SSR
- Migrate to standalone components
- Follow Angular style guide conventions

## Key Concepts
Performance optimization in Angular spans build-time and runtime techniques. AOT compilation, tree shaking, and lazy loading reduce what the browser must download. OnPush change detection, pure pipes, trackBy, and virtual scrolling minimize unnecessary DOM updates and rendering work. Server-side rendering improves perceived load time and SEO by delivering pre-rendered HTML. Following the Angular style guide and adopting standalone components keeps codebases maintainable and aligned with the framework's direction.
