# Components and Templates

## Learning Objectives
- Define components using the @Component decorator with selector, template, and styles
- Choose between inline and external templates and styles
- Apply all four forms of data binding: interpolation, property binding, event binding, and two-way binding
- Use template reference variables to access DOM elements and component instances
- Project content into components with ng-content
- Query child elements and components with ViewChild and ViewChildren

## Key Concepts
Components are the fundamental building blocks of an Angular application. Each component combines an HTML template, CSS styles, and a TypeScript class that controls behavior. Data binding connects the component class to its template, enabling dynamic rendering and user interaction. Content projection with ng-content allows parent components to pass markup into child component templates, while ViewChild and ViewChildren provide programmatic access to child elements.

## Topics Covered
- @Component decorator (selector, template, styles)
- Inline vs external templates
- Data binding (interpolation {{}}, property binding [], event binding (), two-way binding [()])
- Template reference variables (#ref)
- ng-content and content projection
- ViewChild/ViewChildren
