# Routing

## Learning Objectives
- Configure the Angular Router with route definitions
- Navigate between views using routerLink and programmatic navigation
- Extract route parameters, query parameters, and fragments
- Organize features with child and nested routes
- Implement lazy loading with loadChildren and loadComponent
- Protect routes with guards (canActivate, canDeactivate, resolve, canMatch)
- Apply preloading strategies for optimized lazy loading

## Key Concepts
The Angular Router enables single-page application navigation by mapping URL paths to components. Route configuration defines the relationship between URLs and the components that render them. Route parameters and query parameters allow passing data through the URL. Lazy loading defers the loading of feature modules or standalone components until they are needed, improving initial load time. Route guards control access to routes by running logic before navigation completes.
