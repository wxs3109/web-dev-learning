# State Management

## Learning Objectives
- Manage component-level state and lift state into services
- Implement a BehaviorSubject-based store pattern for simple state management
- Understand NgRx architecture including store, actions, reducers, effects, and selectors
- Use Angular Signals (signal, computed, effect) for reactive state
- Build signal-based state management solutions
- Evaluate when to use each state management approach

## Key Concepts
State management governs how application data is stored, updated, and shared across components. Simple applications can manage state through component properties and shared services backed by BehaviorSubject. As complexity grows, NgRx provides a Redux-inspired architecture with a single store, immutable state, and explicit actions. Angular Signals offer a newer, lighter-weight reactive primitive that integrates directly with change detection, providing a middle ground between manual service-based approaches and full NgRx.

## Topics Covered
- Component state vs service state
- BehaviorSubject store pattern
- NgRx overview (store, actions, reducers, effects, selectors)
- Angular Signals (signal, computed, effect)
- Signal-based state management
- When to use which approach
