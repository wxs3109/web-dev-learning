# Forms

## Learning Objectives
- Build template-driven forms using FormsModule, ngModel, ngForm, and ngModelGroup
- Build reactive forms using ReactiveFormsModule, FormControl, FormGroup, FormArray, and FormBuilder
- Apply built-in validators and create custom synchronous and asynchronous validators
- Construct dynamic forms that add or remove controls at runtime
- Implement cross-field validation
- Display validation errors to users

## Key Concepts
Angular provides two complementary approaches to handling user input through forms. Template-driven forms rely on directives in the template to create and manage form controls, making them suitable for simple scenarios. Reactive forms define the form model explicitly in the component class, providing greater control, testability, and scalability. Both approaches support validation, with built-in validators for common rules and the ability to create custom validators for complex business logic.
