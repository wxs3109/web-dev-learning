# HTTP Client

## Learning Objectives
- Set up HttpClientModule and inject HttpClient into services
- Perform GET, POST, PUT, and DELETE requests with typed responses
- Handle HTTP errors using catchError and retry operators
- Create HTTP interceptors for cross-cutting concerns like authentication and logging
- Transform requests and responses
- Track upload and download progress with progress events

## Key Concepts
Angular's HttpClient provides a streamlined API for communicating with backend services over HTTP. It returns Observables, integrating naturally with Angular's reactive patterns. Typed responses give compile-time safety when working with API data. HTTP interceptors sit in the request/response pipeline, enabling centralized handling of authentication tokens, logging, caching, and error recovery without modifying individual service calls.

## Topics Covered
- HttpClientModule setup
- GET/POST/PUT/DELETE requests
- Typed responses
- Error handling (catchError, retry)
- HTTP interceptors (auth tokens, logging, error handling)
- Request/response transformation
- Progress events
