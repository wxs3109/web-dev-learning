# Getting Started

## Learning Objectives
- Use the Angular CLI to create, serve, and generate project artifacts
- Understand the default project structure and key configuration files
- Explain the role of NgModule and the @NgModule decorator
- Distinguish between NgModule-based and standalone component approaches
- Identify Angular versioning conventions and how they affect upgrades

## Key Concepts
The Angular CLI is the primary tool for scaffolding and managing Angular projects. Every new project follows a standard layout with src/app holding application code, angular.json defining workspace configuration, and main.ts bootstrapping the application. NgModule organizes related components, directives, pipes, and services into cohesive blocks, while standalone components offer a lighter alternative that removes the need for NgModule declarations.

## Topics Covered
- Angular CLI (ng new, ng serve, ng generate)
- Project structure walkthrough (src/app, angular.json, main.ts)
- NgModule basics (@NgModule decorator, declarations, imports, providers, bootstrap)
- Standalone components intro
- Angular versioning
