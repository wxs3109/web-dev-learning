# Sass Preprocessor

## Learning Objectives
- Understand the difference between SCSS and indented Sass syntax
- Use Sass variables, nesting, and operators to write concise stylesheets
- Organize code with partials and the @use/@forward module system
- Create reusable style patterns with mixins and functions
- Apply @extend for selector inheritance and understand its trade-offs

## Key Concepts
Sass is a CSS preprocessor that adds programming features like variables, nesting, mixins, and functions to standard CSS. It compiles down to regular CSS and helps manage complexity in larger projects. While native CSS has adopted features like custom properties, Sass remains valuable for its module system, mixins, and build-time logic. This module covers the SCSS syntax and modern Sass features.

## Topics Covered
- SCSS syntax (superset of CSS) vs. indented Sass syntax
- Sass variables with `$variable-name` syntax
- Nesting rules and the `&` parent selector
- Partials: files prefixed with `_` for modular organization
- The `@use` rule for importing modules with namespaces
- The `@forward` rule for re-exporting from index files
- Mixins with `@mixin` and `@include`, including parameters and default values
- `@extend` and placeholder selectors (`%placeholder`)
- Built-in modules: `sass:math`, `sass:color`, `sass:list`, `sass:map`, `sass:string`
- Custom functions with `@function` and `@return`
- Control flow: `@if`, `@else`, `@each`, `@for`, `@while`
- Operators for math, comparison, and string concatenation
- Compiling Sass: CLI tools and build tool integration
