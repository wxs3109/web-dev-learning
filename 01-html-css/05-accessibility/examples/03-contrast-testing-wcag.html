Accessibility Testing Workflow

Color Contrast (WCAG AA)

Good contrast: dark background with white text.

Poor contrast: light gray text on light background.

Awful contrast: even lighter gray text on light background.

WCAG AA generally requires 4.5:1 contrast for normal text.

Screen Reader Testing

  • Windows: NVDA or Narrator
  • macOS/iOS: VoiceOver
  • Check heading order, landmarks, link text, and form labels

DevTools Audits

  • Run Lighthouse accessibility audit in browser DevTools
  • Use axe extension for additional automated checks
  • Always manually verify keyboard and screen reader behavior

WCAG 2.1 Principles (POUR)

  • Perceivable: information must be presentable to users
  • Operable: interface must be usable with keyboard and controls
  • Understandable: content and behavior must be clear
  • Robust: content should work across browsers and assistive tech
