# CSS Architecture

## Learning Objectives
- Apply the BEM naming convention to create predictable, maintainable class names
- Understand the principles of OOCSS and how to separate structure from skin
- Organize stylesheets using the SMACSS categorization model
- Evaluate the utility-first CSS approach and its trade-offs
- Use CSS Modules for locally scoped styles in component-based projects
- Structure stylesheets for large-scale applications

## Key Concepts
As projects grow, CSS can become difficult to maintain without a clear organizational strategy. CSS architecture methodologies provide conventions for naming, structuring, and scaling stylesheets. No single approach is universally best; the right choice depends on team size, project complexity, and tooling. This module surveys the most widely used methodologies and practical strategies for keeping CSS manageable.

## Topics Covered
- The problems of scaling CSS: specificity wars, naming collisions, dead code
- BEM (Block Element Modifier): naming pattern, examples, and benefits
- OOCSS (Object-Oriented CSS): separating structure from skin, container from content
- SMACSS (Scalable and Modular Architecture for CSS): base, layout, module, state, theme categories
- Utility-first CSS: the Tailwind CSS approach, composition over abstraction
- Pros and cons of utility-first vs. component-based CSS
- CSS Modules: locally scoped class names, composition, and build tool setup
- File organization strategies: by feature, by type, index files
- Style linting with Stylelint for enforcing conventions
- Strategies for removing unused CSS (PurgeCSS, tree shaking)
- Combining methodologies: using BEM within a SMACSS structure
