# Forms and Validation

## Learning Objectives
- Build functional HTML forms with appropriate input types
- Associate labels with form controls for accessibility
- Group related fields using fieldsets and legends
- Apply native browser validation with attributes like required, pattern, min, and max
- Handle form submission and understand the default behavior

## Key Concepts
Forms are the primary way users interact with and submit data on the web. HTML provides a rich set of input types and built-in validation attributes that reduce the need for JavaScript. This module covers how to build forms that are usable, accessible, and validate user input before submission.

## Topics Covered
- The `<form>` element and its action/method attributes
- Text-based inputs: `text`, `email`, `password`, `url`, `tel`, `search`
- Numeric and date inputs: `number`, `range`, `date`, `datetime-local`, `time`
- Selection inputs: `checkbox`, `radio`, `<select>`, `<datalist>`
- Other inputs: `file`, `color`, `hidden`
- `<textarea>` for multi-line text
- `<label>` elements and the `for` attribute
- `<fieldset>` and `<legend>` for grouping related controls
- Native validation attributes: `required`, `pattern`, `minlength`, `maxlength`, `min`, `max`, `step`
- The `:valid`, `:invalid`, and `:placeholder-shown` pseudo-classes
- Form submission behavior and the `submit` event
- `<button>` vs. `<input type="submit">`
