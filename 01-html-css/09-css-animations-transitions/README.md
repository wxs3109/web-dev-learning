# CSS Animations and Transitions

## Learning Objectives
- Create smooth state changes with CSS transitions
- Build multi-step animations using @keyframes
- Apply 2D and 3D transforms to elements
- Optimize animation performance for smooth rendering
- Respect user motion preferences with prefers-reduced-motion

## Key Concepts
Animations and transitions bring interfaces to life by providing visual feedback, guiding attention, and creating a polished user experience. CSS provides native tools for both simple state transitions and complex keyframe animations. This module covers how to animate responsibly with an emphasis on performance and accessibility.

## Topics Covered
- CSS transitions: `transition-property`, `transition-duration`, `transition-timing-function`, `transition-delay`
- Shorthand `transition` syntax
- Common timing functions: `ease`, `ease-in`, `ease-out`, `ease-in-out`, `linear`, `cubic-bezier()`
- Properties that can and cannot be transitioned
- `@keyframes` rule: defining animation steps with percentages or from/to
- Animation properties: `animation-name`, `animation-duration`, `animation-timing-function`, `animation-delay`, `animation-iteration-count`, `animation-direction`, `animation-fill-mode`, `animation-play-state`
- 2D transforms: `translate()`, `rotate()`, `scale()`, `skew()`
- 3D transforms: `perspective`, `rotateX()`, `rotateY()`, `translateZ()`
- The `transform-origin` property
- Performance: compositing layers, `will-change`, GPU-accelerated properties (`transform`, `opacity`)
- Avoiding layout thrashing with transforms instead of top/left
- The `prefers-reduced-motion` media query for accessibility
