# Responsive Design

## Learning Objectives
- Write media queries to adapt layouts across screen sizes
- Apply a mobile-first design approach to CSS
- Configure the viewport meta tag for proper scaling on mobile devices
- Implement responsive images that serve appropriate sizes and formats
- Use fluid typography with clamp() for smooth text scaling
- Leverage container queries for component-level responsiveness

## Key Concepts
Responsive design ensures that web pages look and function well on any device, from phones to large monitors. Rather than building separate sites for each screen size, responsive techniques adapt a single codebase to all viewports. This module covers the full toolkit for building responsive layouts and components.

## Topics Covered
- The viewport `<meta>` tag and why it is essential
- Media queries: `@media`, `min-width`, `max-width`, `prefers-color-scheme`, `prefers-reduced-motion`
- Mobile-first vs. desktop-first strategies
- Breakpoint selection and common breakpoint ranges
- Responsive images with `srcset`, `sizes`, and the `<picture>` element
- Fluid typography using `clamp()` for min/preferred/max font sizes
- Container queries (`@container`) for component-scoped responsiveness
- Container query units: `cqw`, `cqh`, `cqi`, `cqb`
- Responsive layout patterns: mostly fluid, column drop, layout shifter, off-canvas
- Testing responsiveness with browser DevTools device mode
