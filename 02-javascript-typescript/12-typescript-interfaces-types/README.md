# TypeScript Interfaces & Types

## Learning Objectives
- Understand the differences and trade-offs between interfaces and type aliases
- Use optional and readonly properties in type definitions
- Extend interfaces for composition and reuse
- Define index signatures for dynamic property access
- Write function type signatures
- Create mapped types and conditional types
- Use template literal types for string pattern enforcement
- Apply built-in utility types: Partial, Required, Pick, Omit, Record, Exclude, Extract, ReturnType

## Key Concepts
TypeScript's type system goes far beyond basic annotations. Interfaces and type aliases allow you to define complex shapes for objects, functions, and data structures. Advanced features like mapped types, conditional types, and utility types enable you to express sophisticated type relationships and reduce duplication in type definitions.
